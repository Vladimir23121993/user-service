# User Service (Spring Boot + REST API + Kafka + Swagger + HATEOAS)

Микросервис для управления пользователями с REST API, интеграцией с Apache Kafka, отправкой событий при создании/удалении пользователя, Swagger-документацией и HATEOAS-ссылками для навигации по API.

Проект построен на слоистой архитектуре:

Основные слои:
- entity — описание пользователя (id, имя, email, возраст, дата создания)
- dto — объекты для передачи данных (запрос/ответ), чтобы не возвращать сущность напрямую
- repository — работа с базой через Spring Data JPA
- service — бизнес-логика и проверки (например, что email содержит @)
- controller — REST контроллер, принимает запросы и возвращает JSON
- mapper — конвертирует Entity в DTO и обратно
- kafka — отправка событий в Apache Kafka при создании/удалении пользователя
- model — модель с поддержкой HATEOAS (ссылки _links)
- config — настройки Swagger и CORS

Дополнительно:
- console — консольное меню для быстрого тестирования (CommandLineRunner)

Возможности программы (REST API):

- POST /api/users — создать нового пользователя
- GET /api/users — получить всех пользователей
- GET /api/users/{id} — найти пользователя по ID (с HATEOAS-ссылками)
- PUT /api/users/{id} — обновить данные пользователя
- DELETE /api/users/{id} — удалить пользователя
- GET /api/users/email?email= — найти пользователя по email
- GET /api/users/search?name= — найти пользователей по имени (частичное совпадение)

Kafka:

- При создании пользователя отправляется событие CREATED в топик `user-events-2`
- При удалении пользователя отправляется событие DELETED в топик `user-events-2`

Swagger (документация API):

- Доступен по адресу: `http://localhost:8080/swagger-ui/index.html`
- Позволяет тестировать API прямо из браузера
- Содержит описание всех эндпоинтов, параметров и ответов

HATEOAS (ссылки для навигации):

- В ответах на запросы присутствуют `_links` с ссылками на связанные ресурсы
- Позволяет клиенту навигировать по API без документации

Тесты:

Интеграционные тесты проверяют загрузку контекста Spring и работоспособность приложения
Используют @SpringBootTest для поднятия полного контекста
